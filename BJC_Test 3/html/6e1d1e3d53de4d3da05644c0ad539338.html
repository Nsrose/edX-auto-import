Drawing Conclusions

Suddenly, your crime-fighter colleagues come in and have a hunch: Maybe there was more than one perpetrator? The messages were collected in a very short time and no major movement was observed. So they ask you to confirm or deny the hypothesis that the messages come from more than one source and if so from how many.

Clustering

Clearly, all coordinates have different values. So we have to find out if certain points are closer to other points and if that's a pattern. In other words: Can we find clusters and how many?

In order to do that, we need two main components:

  • A method to determine which points are close to each other vs which points are far away from each other. This is called a distance metric.  An easy to use metric is the Euclidean Distance d, which for two points p and q with coordinates p1 and q1 and p2 and q2 is given as
  • A method to determine if a point belongs to a cluster (a set of points) or not, based on the distance metric. A cluster can be defined in various ways but one easy way is to use the minimum distance from each point in the cluster. A certain point then belongs to a cluster if the distance between this point and the point with the minimum distance is smaller than a certain value.

Drawing Conclusions

Eye-balling and visualization is good but numbers are better. So let's put it all together and find out how well your clustering did. The goal is to come up with a quality statement like: X% of the points lie withing Y clusters. Aiming at high values of X would provide evidence for the validation of our hypothesis.
So what we need is a function that takes as an input the cluster data, the clusters, and a distance and returns how many points lie within in.
