Make the Game Interactive

At last you're ready to make this a real, interactive game. Then, if you're still game, you can improve the strategy.

First, be sure you've turned in your work so far.

This page and the next are optional! But you really should find it all within your ability by now.

We're just going to sketch out a structure, not go through it in detail.

1. Ask the user if s/he wants to play first (X) or second (O).

2. Draw the board on the stage.

3. If the computer is going first, move in square 5.

4. Then comes a forever loop:

a. If there are no free squares, declare a tie and stop.

b.Tell the user to move, and detect a mouse click in one of the squares as the move.

c. Update the position.

d. If the user has won, say so and stop.

e. Use the ttt block to choose the computer's move.

f. Update the position.

g. If the computer has won, say so and stop.

That's it. You'll want to write helpers such as draw-x and draw-o that take a square number as input and draw two crossed lines or a circle on the stage in the right place, and tie? and winner? to handle steps 4a, 4d, and 4g.
